tcc 的词法分析(二)#

宏只处理 define

做法是如果遇到宏，就跳转过去读，作为最新的 token 然后返回

结构#

符号表

  • char *idtable 是一个字符数组，存储着每一个符号。符号之间用 ‘\0’ 分割
  • char *idptr, *idlast 辅助指针
    • idptr 指向字符数组第一个空位置
    • idlast 指向符号表中最后的一个符号(插入新符号的时候，重置该指针)

tinycc 在处理文本的时候，除了运算符和数字，都视为符号

返回的时候 token 表明了符号位于符号表的位置（第几个符号）

  • 从 256 开始记的
  • 前面的属于关键字
  • 因为字符直接使用数值作为类型，因此根据 ascii 从 256 开始是符号(包括关键字)

步骤：

  1. 只要第一个判断属于符号，进入符号的状态机，逐个字符插入符号表
  2. 比较判断是否为关键字，如果是的话，保持 idptr 不变，同时更新 tok
  3. 如果不是关键字的话，更新 idptr
  4. 判断该符号是否位于宏替换的范围，如果是的话，不要立即返回
  5. 记录当前输入流读取位置，到宏栈中（维护了一个栈）
  6. 设置输入流的位置到宏替换的位置
  7. 重新调用 next() 来读取替换的符号
  8. 直到行尾，替换结束，根据宏栈的地址，跳转回来（允许嵌套宏）

宏

步骤：

  1. 如果当前符号是 #, 进入分支
  2. 取下一个符号，如果下一个符号就是 define 关键字（固定 token 类型）
  3. 读取下一个符号到符号表中，同时更新 vat 和 vac, 在该符号位置记录了符号类型和数值
  • 对于 define 的符号类型就是 VT_DEFINE, 数值就是文件流的位置，就是现在
  1. 一直跳过字符不再读取了，这部分是宏替换的内容。直到读取完换行符为止

宏替换时：

  1. 如果当前位于宏替换（宏栈内有数据），且读取到了换行符
  2. 跳转回去（更新输入流的位置）
